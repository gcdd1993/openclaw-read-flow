# Pipeline Contracts

## 阶段输入输出总表

### 1. 信息源接入

输入：

- 站点、账号、栏目、仓库、页面或现有 feed

输出：

- 统一后的 feed 入口
- 源注册表条目
- 源分组结果

### 2. Digest 预处理

输入：

- FreshRSS 未读列表或其他聚合池未读候选

输出：

- 去重后的候选条目
- 同事件聚类
- 正文抓取状态
- 摘要和初排序
- FreshRSS 原始导出（若适用）

### 3. Daily Review 精选

输入：

- Digest 候选文档

输出：

- 栏目化日报
- 同事件合并结果
- 今日重点与后续跟踪建议

### 4. 知识沉淀

输入：

- 人工确认值得精读或留存的内容

输出：

- 知识卡片
- 周刊素材
- 主题长期积累条目

### 5. 反馈与画像

输入：

- 点开、读完、收藏、沉淀、负反馈等行为信号

输出：

- 反馈事件记录
- 兴趣画像摘要
- Digest / Daily Review 的弱排序提示

## 总控判定规则

- 缺少上游产物时，优先补上游。
- 只有在输入足够稳定时，才进入下游编辑或沉淀。
- 与长期价值有关的动作，默认需要人工确认。
- 画像只能影响排序强弱，不能替代内容价值判断。
